**2. Update `packages/extension/entrypoints/content/messaging.ts`:**

Add imports:
- `import { getUniqueSelector } from './element-detector'`
- `import { queryFiber } from './fiber-bridge'`
- `import { resolveSourceLocation, findComponentSource } from './source-resolver'`

Change `sendElementSelection(el: Element): void` to `async sendElementSelection(el: Element): Promise<void>`.

After building the base payload object (tagName, className, id, xpath, boundingRect — keep this exactly as-is), add an enrichment block wrapped in try/catch:

```
try {
  const selector = getUniqueSelector(el);
  const fiberResult = await queryFiber(selector);

  if (fiberResult.componentName) {
    payload.componentName = fiberResult.componentName;
  }
  if (fiberResult.parentChain.length > 0) {
    payload.parentChain = fiberResult.parentChain;
  }

  // Source resolution: _debugSource (React 17/18) or heuristic (React 19)
  if (fiberResult.debugSource) {
    const resolved = await resolveSourceLocation(
      fiberResult.debugSource.fileName,
      fiberResult.debugSource.lineNumber,
      0
    );
    if (resolved) {
      payload.sourceFile = resolved.source;
      payload.sourceLine = resolved.line;
      payload.sourceColumn = resolved.column;
    } else {
      payload.sourceFile = fiberResult.debugSource.fileName;
      payload.sourceLine = fiberResult.debugSource.lineNumber;
    }
  } else if (fiberResult.componentName) {
    const found = await findComponentSource(fiberResult.componentName);
    if (found) {
      payload.sourceFile = found.source;
      payload.sourceLine = found.line;
      payload.sourceColumn = found.column;
    }
  }
} catch {
  // Enrichment failed — send basic payload without component info
}
```

The `chrome.runtime.sendMessage(payload).catch(() => {})` call at the end stays the same.

Update the payload type to include the optional fields. Since the payload is built as an object literal, add type annotation or extend the inline object type to include `componentName?: string`, `parentChain?: string[]`, `sourceFile?: string`, `sourceLine?: number`, `sourceColumn?: number`. The simplest approach: declare `const payload: Record<string, unknown> & { type: 'element_selection' }` or use a typed interface matching `ElementSelection` from shared schemas.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Fiber bridge → messaging | Data from fiber bridge (originated in untrusted page context) flows into the extension message payload |
| Messaging → sidebar | Content script sends enriched payload to sidebar via chrome.runtime.sendMessage |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-06 | Tampering | messaging.ts payload | mitigate | Wrap fiber/source enrichment in try/catch — malformed data from bridge causes enrichment skip, not crash. Sidebar receives basic DOM payload. |
| T-04-07 | Information Disclosure | App.tsx path display | mitigate | Truncate displayed file paths to project-relative segments. Strip absolute filesystem prefixes. Never show full system paths in sidebar UI. |
| T-04-08 | Spoofing | chrome.runtime.sendMessage | accept | Messages between content script and sidebar use Chrome's internal messaging — cannot be intercepted by page scripts. Low risk, no mitigation needed. |
</threat_model>
