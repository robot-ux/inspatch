# Phase 8 — UI Design Contract

> Visual and interaction contract for the full dark theme redesign of the Inspatch Chrome extension sidebar. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Reference products:** DevLens Pro, VisBug, Chrome DevTools dark theme
**Sidebar width:** ~320px (Chrome side panel)
**Nature:** Full visual overhaul — all functionality is built, this phase is purely visual/UX.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (hand-crafted CSS custom properties) |
| Preset | not applicable |
| Component library | none (custom components, no shadcn — sidebar is too specialized) |
| Icon library | none (inline SVG + Unicode glyphs, matching current approach) |
| UI font | `system-ui, -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif` |
| Code font | `'SFMono-Regular', Consolas, 'Liberation Mono', Menlo, Courier, monospace` |

**Rationale:** A Chrome extension sidebar at ~320px with 9 specialized states does not benefit from a component library. CSS custom properties give full control over the dark theme tokens, and Tailwind v4's CSS-first config integrates them natively via `@theme`.

---

## CSS Custom Properties

All tokens are defined as CSS custom properties on `:root` in `style.css` and consumed via Tailwind's `@theme` directive. This is the single source of truth.

```css
:root {
  /* Surfaces */
  --ip-bg-primary: #0F172A;          /* slate-900 — dominant surface */
  --ip-bg-secondary: #1E293B;        /* slate-800 — cards, sections */
  --ip-bg-tertiary: #334155;         /* slate-700 — elevated elements */
  --ip-bg-card: rgba(30, 41, 59, 0.6); /* slate-800 at 60% — card backgrounds */
  --ip-bg-input: rgba(15, 23, 42, 0.8); /* input fields */

  /* Borders */
  --ip-border-subtle: rgba(148, 163, 184, 0.1);  /* slate-400/10 — default */
  --ip-border-muted: rgba(148, 163, 184, 0.2);   /* slate-400/20 — hover */
  --ip-border-accent: rgba(99, 102, 241, 0.4);    /* indigo-500/40 — focus */

  /* Text */
  --ip-text-primary: #F1F5F9;        /* slate-100 */
  --ip-text-secondary: #94A3B8;      /* slate-400 */
  --ip-text-muted: #64748B;          /* slate-500 */
  --ip-text-accent: #A5B4FC;         /* indigo-300 */

  /* Brand gradient */
  --ip-gradient-start: #3B82F6;      /* blue-500 */
  --ip-gradient-end: #8B5CF6;        /* violet-500 */
  --ip-accent-solid: #6366F1;        /* indigo-500 — fallback when gradient not possible */

  /* Semantic */
  --ip-success: #22C55E;             /* green-500 */
  --ip-success-muted: rgba(34, 197, 94, 0.15);
  --ip-warning: #F59E0B;             /* amber-500 */
  --ip-warning-muted: rgba(245, 158, 11, 0.15);
  --ip-error: #EF4444;               /* red-500 */
  --ip-error-muted: rgba(239, 68, 68, 0.15);
  --ip-info: #3B82F6;                /* blue-500 */
  --ip-info-muted: rgba(59, 130, 246, 0.15);

  /* Radii */
  --ip-radius-sm: 6px;
  --ip-radius-md: 8px;
  --ip-radius-lg: 12px;

  /* Shadows */
  --ip-shadow-card: 0 1px 3px rgba(0, 0, 0, 0.3), 0 1px 2px rgba(0, 0, 0, 0.2);
  --ip-shadow-glow-accent: 0 0 12px 2px rgba(99, 102, 241, 0.25);
```

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, tight element pairs |
| sm | 8px | Compact element spacing, button padding-y |
| md | 12px | Section internal padding (compact sidebar density) |
| lg | 16px | Card padding, default content spacing |
| xl | 24px | Section gaps, major content separation |
| 2xl | 32px | Layout-level spacing |
| 3xl | 48px | Hero/empty state vertical centering gaps |

**Project override:** 12px (3×4) added to the standard set {4, 8, 16, 24, 32, 48, 64} — the 8→16 gap is too coarse for a ~320px sidebar panel where compact density is essential. All values remain 4px-grid-aligned.

Exceptions: border-radius uses its own token scale (4px, 8px, 12px) — not spacing tokens but noted for consistency.

## Typography

| Role | Size | Weight | Line Height | Font Stack | Usage |
|------|------|--------|-------------|------------|-------|
| Caption | 11px | 400 (regular) | 1.45 | system-ui | Hints, metadata, secondary info, timestamps |
| Body | 13px | 400 (regular) | 1.5 | system-ui | Default UI text, descriptions, status messages |
| Label | 13px | 600 (semibold) | 1.5 | system-ui | Button labels, section headers, field labels |
| Heading | 16px | 600 (semibold) | 1.25 | system-ui | Card titles, state headings |
| Code | 12px | 400 (regular) | 1.4 | SFMono-Regular, Consolas, monospace | File paths, component names, xpaths, CSS classes, diffs, code blocks |

**Rules:**
- Never use `font-weight: bold` (700) — max weight is 600 (semibold)
- Code font is used for ALL technical data: tags, classes, xpaths, component names, file paths, diffs, server commands
- UI font is used for all labels, descriptions, status messages, CTAs

- Color/background transitions: `150ms ease`

- Layout transitions: `300ms ease-out`

## State Transition Map

```
[Not Localhost] ────────────────────────────────── (tab URL check)
       │
       ▼ (is localhost)
[Disconnected] ◄──── ws close/error
       │
       ▼ ws connected
[Connected Idle] ◄──── element cleared / inspect stopped
       │
       ▼ "Start Inspect" clicked
[Inspecting] ──── "Stop Inspect" → [Connected Idle]
       │
       ▼ element selected (Alt+Click)
[Element Selected] ◄──── "Inspect Again" cycles back to [Inspecting]
       │                     │
       │  (no source file)   │
       │  └─ [No Source Warning] shown above input
       │
       ▼ "Send Change" clicked
[Processing] ──── status_update messages cycle through:
       │           queued → analyzing → locating → generating → applying
       │
       ├──► [Result Success] ──── "Inspect Again" → [Inspecting]
       └──► [Result Error] ──── "Try Again" → clears result, stays on [Element Selected]
```

## Implementation Notes

### CSS Custom Properties Strategy
1. Define all `--ip-*` tokens in `:root` block in `style.css`
2. Register tokens with Tailwind v4 via `@theme` directive for utility class access
3. Components consume tokens via Tailwind utilities (e.g., `bg-[var(--ip-bg-card)]`) or direct CSS variable references
4. No light theme variant needed — extension is dark-only

### File Changes Expected
| File | Change |
|------|--------|
| `style.css` | Add CSS custom properties, update shimmer opacity, register `@theme` tokens |
| `App.tsx` | Replace all light theme classes with dark theme token references |
| `ProcessingStatus.tsx` | Dark theme status cards, update color mappings |
| `ChangeInput.tsx` | Dark theme input area, gradient send button |

### Scrollbar Styling
- Scrollbar track: `--ip-bg-primary`
- Scrollbar thumb: `--ip-bg-tertiary`
- Scrollbar thumb hover: `--ip-text-muted`
- Width: 4px (thin scrollbar for compact sidebar)
