Create `packages/extension/entrypoints/sidepanel/hooks/useScreenshot.ts` with:

1. A `loadImage(src: string): Promise<HTMLImageElement>` helper that returns a promise resolving when the image loads (standard `new Image()` + onload/onerror pattern).

2. A `cropScreenshot(fullDataUrl: string, rect: { x: number; y: number; width: number; height: number }, dpr: number): Promise<string>` function that:
   - Loads the full screenshot via `loadImage()`
   - Creates an offscreen `document.createElement('canvas')`
   - Computes source coordinates scaled by dpr: `sx = rect.x * dpr`, `sy = rect.y * dpr`, `sw = rect.width * dpr`, `sh = rect.height * dpr`
   - Clamps to image bounds: `clampedW = Math.min(sw, img.width - sx)`, `clampedH = Math.min(sh, img.height - sy)`. Also clamp sx/sy to be >= 0.
   - Applies max dimension cap: if either `clampedW` or `clampedH` exceeds 1920, scale both down proportionally
   - Sets canvas dimensions to the (potentially scaled) output dimensions
   - Draws the cropped region with `ctx.drawImage(img, sx, sy, clampedW, clampedH, 0, 0, outputW, outputH)`
   - Smart format selection per D-02: if `rect.width <= 200 && rect.height <= 200`, use `canvas.toDataURL('image/png')`. Otherwise use `canvas.toDataURL('image/jpeg', 0.80)`.
   - Returns the data URL string

3. A `useScreenshot()` hook exported as the default export that:
   - Maintains state: `screenshotUrl: string | null`, `isCapturing: boolean`, `error: string | null`
   - Exposes a `capture(boundingRect: { x: number; y: number; width: number; height: number }, devicePixelRatio: number): Promise<string | null>` async function (wrapped in `useCallback`)
   - `capture()` implementation:
     a. Sets `isCapturing = true`, clears error
     b. Calls `chrome.tabs.captureVisibleTab(undefined, { format: 'png', quality: 100 })` — always capture as PNG for lossless source, then the crop step handles format selection
     c. Calls `cropScreenshot(dataUrl, boundingRect, devicePixelRatio)`
     d. Sets `screenshotUrl` to the result
     e. Sets `isCapturing = false`
     f. Returns the cropped data URL
     g. On error: sets `error` to the message, `isCapturing = false`, returns null
   - Exposes a `clear()` function that resets `screenshotUrl` to null
   - Returns `{ screenshotUrl, isCapturing, error, capture, clear }`

Import `createLogger` from `@inspatch/shared` and create a logger tagged `'screenshot'` for error logging.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-04 | Denial of Service | Screenshot payload size | mitigate | Max dimension cap at 1920px in useScreenshot.ts. Smart format: JPEG 80% for large elements reduces 3-8MB PNG to 200-500KB. |
| T-05-05 | Tampering | description field | mitigate | Zod `.min(1)` validation. Description is passed to Claude CLI as prompt text, never rendered as HTML. Server-side validation in Phase 6. |
| T-05-06 | Tampering | screenshotDataUrl | accept | Data URL comes from captureVisibleTab (trusted Chrome API) + canvas crop. Not user-supplied. Validated as string by Zod. |
| T-05-07 | Information Disclosure | captureVisibleTab scope | accept | Scoped to localhost via host_permissions. Cannot capture non-localhost tabs. Single-user local tool — no privacy concern. |
| T-05-08 | Denial of Service | captureVisibleTab rate limit | mitigate | Capture once on element selection, cache in React state. No re-capture on re-render. Chrome limit is 2 calls/second. |
</threat_model>
