   - `state: InspectState = 'idle'`
   - `currentTarget: Element | null = null`
   - `selectedElement: Element | null = null`
   - `rafId: number | null = null`

3. `start()` method:
   - Guard: `if (this.state !== 'idle') return`
   - Set `this.state = 'inspecting'`
   - Call `mountOverlay(this.host)`
   - Call `showLayers(this.layers)`
   - Bind event handlers and store references:
     - `this.boundHandlers.mousemove = this.handleMouseMove.bind(this)`
     - `this.boundHandlers.click = this.handleClick.bind(this)`
     - `this.boundHandlers.keydown = this.handleKeyDown.bind(this)`
   - Register DOM event listeners using capture phase (third arg `true`):
     - `document.addEventListener('mousemove', this.boundHandlers.mousemove, true)`
     - `document.addEventListener('click', this.boundHandlers.click, true)`
     - `document.addEventListener('keydown', this.boundHandlers.keydown, true)`
   - Start rAF loop: `this.startRafLoop()`

4. `stop()` method:
   - Set `this.state = 'idle'`
   - Call `this.stopRafLoop()`
   - Call `hideLayers(this.layers)`
   - Call `unmountOverlay(this.host)`
   - Remove event listeners: `document.removeEventListener('mousemove', ...)`, `document.removeEventListener('click', ...)`, `document.removeEventListener('keydown', ...)` all with capture `true`
   - Reset: `this.currentTarget = null`, `this.selectedElement = null`
   - Notify sidebar that inspect mode has stopped: `chrome.runtime.sendMessage({ type: 'inspect-stopped' }).catch(() => {})` — catch silently if no listener is ready (sidebar may not be open). This enables the sidebar (Plan 02) to reset its button state when Escape is pressed on the page.

5. `handleMouseMove(e: MouseEvent)` (private):
   - `const target = getElementAtPoint(e.clientX, e.clientY, this.host)`
   - If `target === null` or `target === this.currentTarget`, return
   - `this.currentTarget = target`
   - `positionOverlayLayers(target, this.layers)`

6. `handleClick(e: MouseEvent)` (private):
   - ALWAYS suppress all clicks during inspect mode (per D-08: "Normal clicks are suppressed during inspect mode via preventDefault"):

   - If `this.currentTarget === null`, return

   - `positionOverlayLayers(this.selectedElement, this.layers)` (persist highlight position)
   - `this.onSelect(this.selectedElement)` — triggers sendElementSelection via callback
   - `this.stopRafLoop()` — highlight is now static on selected element
   - Remove mousemove listener only — stop hover updates while selected, keep click and keydown active

7. `handleKeyDown(e: KeyboardEvent)` (private):
   - If `e.key === 'Escape'`: call `this.stop()`

8. `startRafLoop()` (private):

     - If `this.state === 'idle'`, return (stop looping)
     - If `this.currentTarget !== null`: `positionOverlayLayers(this.currentTarget, this.layers)` — handles scroll/resize reposition per D-03 and Research Pitfall 3

   - Kick off with `this.rafId = requestAnimationFrame(loop)`

9. `stopRafLoop()` (private):
   - If `this.rafId !== null`: `cancelAnimationFrame(this.rafId)`, `this.rafId = null`

```typescript
export default defineContentScript({

      inspectMode.stop();
      cleanupMessaging();
      host.remove();
    });
  },
});
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Content script ↔ page DOM | Content script runs in isolated world but shares DOM with the inspected page |
| Content script ↔ extension messaging | chrome.runtime.onMessage only receives from extension contexts, not page JS |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | overlay-manager.ts host element | accept | Page JS can modify the `<inspatch-overlay>` host element but cannot reach inside the shadow root. Worst case: overlay disappears, user re-enters inspect mode. Low impact for a localhost dev tool. |
| T-02-02 | Info Disclosure | overlay-manager.ts | accept | Page JS can detect `<inspatch-overlay>` in DOM, revealing extension is active. No sensitive data is exposed — element tag name is not secret. Acceptable for a dev tool. |
| T-02-03 | Spoofing | messaging.ts | accept | Page JS cannot send to `chrome.runtime.onMessage` — only extension contexts can. No mitigation needed per Chrome extension security model. |
| T-02-04 | Denial of Service | element-detector.ts | accept | Malicious page with extreme DOM depth could slow `elementFromPoint`. This is a local dev tool where pages are developer-controlled. No mitigation needed. |
</threat_model>
