Helper function to send commands to content script (per D-12):
```typescript
const sendToContentScript = useCallback(async (message: { type: string }) => {
  const [tab] = await chrome.tabs.query({ active: true, lastFocusedWindow: true });
  if (!tab?.id) return;
  return chrome.tabs.sendMessage(tab.id, message);
}, []);
```

useEffect to listen for element_selection from content script (per D-11):
```typescript
useEffect(() => {
  const listener = (message: unknown) => {
    if (message && typeof message === 'object' && 'type' in message) {
      const msg = message as { type: string };
      if (msg.type === 'element_selection') {
        setSelectedElement(message as ElementSelection);
        setSidebarState('selected');
      } else if (msg.type === 'inspect-stopped') {
        setSidebarState('idle');
        setSelectedElement(null);
      }
    }
  };
  chrome.runtime.onMessage.addListener(listener);
  return () => chrome.runtime.onMessage.removeListener(listener);
}, []);
```

Note: Also listen for `'inspect-stopped'` in case Escape was pressed on the page side — the content script messaging.ts sends this when inspect mode stops. If Plan 01 didn't add this message, the sidebar still works correctly from button clicks alone; the Escape key would just not update the sidebar button state. Handle gracefully either way.

Action handlers:
```typescript
const handleStartInspect = useCallback(async () => {
  await sendToContentScript({ type: 'start-inspect' });
  setSidebarState('inspecting');
}, [sendToContentScript]);

const handleStopInspect = useCallback(async () => {
  await sendToContentScript({ type: 'stop-inspect' });
  setSidebarState('idle');
  setSelectedElement(null);
}, [sendToContentScript]);

const handleInspectAgain = useCallback(async () => {
  setSelectedElement(null);
  await sendToContentScript({ type: 'start-inspect' });
  setSidebarState('inspecting');
}, [sendToContentScript]);
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Sidebar ↔ content script | Commands and selection data cross extension context boundaries via chrome.tabs/runtime API |
| Element data → sidebar DOM | Untrusted page element attributes (tagName, className, id, xpath) rendered in sidebar React UI |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05 | Injection | App.tsx element info display | mitigate | React auto-escapes all JSX text content via `{variable}` syntax. Never use `dangerouslySetInnerHTML` to render element data. All fields (tagName, className, id, xpath) are rendered as text nodes only. |
| T-02-06 | Spoofing | App.tsx chrome.runtime.onMessage listener | accept | `chrome.runtime.onMessage` in the sidebar page only receives messages from extension contexts (content scripts, background). Page JS cannot send to this listener per Chrome extension security model. |
</threat_model>
